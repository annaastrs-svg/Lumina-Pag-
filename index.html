El ESP32-C3 Super Mini fue elegido por su tamaño ultra compacto y porque corre el mismo código Arduino que cualquier ESP32 estándar. El bus I2C permite conectar la pantalla y el sensor en solo 2 pines de datos, dejando el resto libre.

ensamblaje

Cableado & Pines

Todo el sistema comparte la alimentación de 3.3V del ESP32. El bus I²C (SDA/SCL) conecta en paralelo la pantalla OLED y el sensor de pulso — sin conflicto porque tienen direcciones I²C distintas (0x3C y 0x57).

GND——GND

GND——GND
GPIO 21——SCK (SCL)

Código & Lógica

El firmware corre en Arduino (framework ESP32). El loop principal lee el sensor, aplica un filtro de plausibilidad, promedia con un búfer circular de 6 muestras y despacha el estado emocional — todo sin usar delay(), para que la pantalla nunca se congele.

filtro BPM
Solo acepta 45–145 BPM. Buffer circular de 6 muestras. Promedio estable.
⏱
buzzer
Non-blocking: millis() en vez de delay(). Pitido de 40ms exactos.

 Loop principal — no bloqueante
void loop() {
  long irValue = particleSensor.getIR();

   Sin dedo → modo standby
  if (irValue < 50000) {
    estadoActual = DESCONECTADO;
    beatAvg = 0;
  } else {
    if (checkForBeat(irValue) == true) {
      long delta = millis() - lastBeat;
      lastBeat = millis();
      beatsPerMinute = 60 / (delta / 1000.0);

       Filtro de plausibilidad: solo valores humanos reales
      if (beatsPerMinute >= 45 && beatsPerMinute <= 145) {
        rates[rateSpot++] = (byte)beatsPerMinute;
        rateSpot %= RATE_SIZE; // Buffer circular

        long total = 0;
        for (byte x = 0; x < RATE_SIZE; x++) total += rates[x];
        beatAvg = total / RATE_SIZE; // Promedio suavizado

         Activar buzzer — tono según estado emocional
        buzzerActivo = true;
        tiempoBuzzerEncendido = millis();
        if (beatAvg > 100) {
          tone(BUZZER_PIN, 1800); // Agudo: alerta
        } else {
          tone(BUZZER_PIN, 1200); // Normal: calmo
        }
      }
    }
    Clasificar estado emocional
    estadoActual = (beatAvg == 0) ? DESCONECTADO
                 : (beatAvg > 100) ? AGITADO : CALMO;
  }

   Apagar buzzer automáticamente a los 40ms
  if (buzzerActivo && (millis() - tiempoBuzzerEncendido > 40)) {
    noTone(BUZZER_PIN);
    buzzerActivo = false;
  }

  actualizarPantalla(); // Siempre re-dibuja
}
Lumina.ino — actualizarPantalla() (lógica de caras) 
void actualizarPantalla() {
  display.clearDisplay();

  // BPM en tamaño grande (zona inferior)
  display.setTextSize(2);
  display.setCursor(0, 48);
  display.print("BPM:");

  switch (estadoActual) {
    case CALMO:
      display.println("Lumina: Calmado");
      // Ojos: ^ ^ alternando con o o cada 500ms
      if (alternarAnimacion) {
        display.print("^"); display.print("^");
      } else {
        display.print("o"); display.print("o");
      }
      break;

    case AGITADO:
      display.println("!Respira profundo!");
      // Ojos bien abiertos, boca tensa
      display.print("O"); display.print("O");
      break;

    case DESCONECTADO:
      display.println("Modo Standby");
      // Ojos dormidos: u u o guiones
      break;
  }
  display.display(); // Flush al panel físico
}

Dificultades & Aprendizajes

Por mas sencillo que sea hay dificultades aveces

  • 1
    El oxímetro roto de mi v.1 no daba señal limpia El primer intento fue rescatar el sensor del oxímetro casero con el pin TX del ESP32. Funcionó parcialmente — leía valores, pero eran erráticos e inconsistentes. NO HABIA MANERA DE LEER BPM confiable. La solución fue rendirme y comprar el módulo MAX30102 de verdad.
  • 2
    BPM con saltos absurdos (40 a 180 en segundos) Sin filtro, cualquier movimiento del dedo o ruido del sensor disparaba lecturas de 190 BPM o caía a 30.
  • 3
    La pantalla se congelaba cuando el buzzer sonaba El uso de delay() para el buzzer bloqueaba el loop entero, haciendo que la OLED dejara de actualizarse. Reescribí el control del buzzer usando millis()
  • 4
    Conflicto de bus I2C entre OLED y MAX30102 Ambos dispositivos comparten SDA y SCL. Al principio, la pantalla y el sensor se interferian. La solución fue inicializar ambos con la dirección I2C correcta (0x3C para OLED, 0x57 para el sensor)
